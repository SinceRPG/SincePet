Feature: Cooldown Reduction Upgrades & Dynamic GUI Stats

**New Features**
- Added `skill_cooldown_formula` to pet upgrades. Players can now upgrade their pets to reduce the cooldown of active and passive MythicMobs abilities based on the upgrade level.
- Added `attack_speed_formula` to pet upgrades. Players can upgrade this to reduce the delay between the pet's auto-attacks.
- The `<skills>` placeholder in the collection menu now dynamically calculates and displays the current skill cooldown based on the player's active cooldown reduction bonus!
- Added new GUI placeholders for upgrade items: `<skill_cooldown_bonus>`, `<next_skill_cooldown_bonus>`, `<attack_speed_bonus>`, `<next_attack_speed_bonus>`.

**Enhancements & Refactoring**
- Removed the hardcoded "All Stats" string. It can now be customized via `LEGACY_ALL` under the `stats` section in `gui.yml`.
- Updated default `gui.yml` to include the new placeholders in the `available` and `maxed` upgrade lore.
- Added example cooldown reduction formulas to `bear.yml` and `dragon.yml` defaults.
- Updated default `messages.yml` collection lore to properly use the multi-stat `<stats>` placeholder instead of the legacy single `<stat>`.
- Fully updated the Markdown wiki (`docs/*.md`) to thoroughly document the `stats` map structure, the new formulas, and the new GUI placeholders.

**Files to Reset / Update (Recommended)**
If you are updating from an older version, please consider regenerating or manually updating the following files:
- `gui.yml` (to add `LEGACY_ALL` translation and the new `<skill_cooldown_bonus>` & `<attack_speed_bonus>` placeholders in `upgrade_items` lore).
- `messages.yml` (to replace `<stat>` with `<stats>` in the `collection.pet_item.lore`).
- `pets/bear.yml` and `pets/dragon.yml` (optional, if you want to see the new formula examples).
